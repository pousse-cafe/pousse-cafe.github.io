Pousse-Café

Making Domain-Driven Design implementation easy

Quick description

Pousse-Café is a Java framework that eases the writing of applications following Domain-Driven Design (DDD) methodology. It relies on the definition of a meta-application described essentially by Domain logic plus related Domain processes. Meta-applications can be integrated in real applications which provide the adapters connecting it to the real world.

Clean

Your domain logic is not crippled by data-representation, event handling or storage-related implementation details. This allows to focus on clarity and readability.

Flexible

Pousse-Café has very few dependencies, reducing dramatically the risk of conflict with other surrounding frameworks. Also, Pousse-Café cohabits well with legacy code, allowing for step-by-step migration to DDD.

Technology agnostic

Pousse-Café meta-applications are written without relying on storage and/or messaging technologies. This allows to postpone technology choices without impacting the capacity to deliver business value. It also eases the migration from one technology to another.

Get started!

  1. Generate a simple app Maven project:

    mvn archetype:generate -B \
    -DarchetypeGroupId=org.pousse-cafe-framework \
    -DarchetypeArtifactId=pousse-cafe-simple-meta-app-archetype \
    -DarchetypeVersion={{ site.latest_release_version }} \
    -DgroupId=test \
    -DartifactId=test \
    -Dversion=1.0.0-SNAPSHOT
  2. Open the project in your favorite IDE or test it already by executing mvn test
  3. Voilà
