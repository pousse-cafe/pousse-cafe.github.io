How it works

  • A Pousse-Café model (i.e. a set of Aggregates and Services) is executed by a Runtime
  • Commands are submitted to the Runtime
  • Commands are handled by Message Listeners defined in Aggregates or their Factory or Repository
  • Aggregates issue Domain Events
  • The set of Message Listeners executed following the submission of a Command defines a Domain Process
  • Aggregates can be grouped in Modules
  • Domain Events can cross Modules borders

See the reference guide for more information.

An Example

There is an example application available in GitHub. See README for instructions about how to run it and play with it. You can also just check-out the code.

Get started!

  1. Generate your first Pousse-Café project (do not forget to replace archetype version with the latest ): 

    mvn archetype:generate -B \
      -DarchetypeGroupId=org.pousse-cafe-framework \
      -DarchetypeArtifactId=pousse-cafe-sample-app-archetype \
      -DarchetypeVersion=${pousse-cafe.version} \
      -DgroupId=test \
      -DartifactId=test \
      -Dversion=0.0.0-SNAPSHOT
  2. Open the project in your favorite IDE
  3. Take the tour of the code
