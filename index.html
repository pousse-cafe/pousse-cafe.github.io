Pousse-Café Framework

Distil knowledge with your code

Business Process Oriented

  • Focus on processes of the domain
  • Defined using DDD primitives
  • Favors scalable implementation

Living Documentation Ready

  • Expert-readable documentation
  • Generated at build time
  • Output in PDF and HTML

Technology Independent

  • Clean separation of concerns
  • Increased flexibility for technology choice
  • Reduced risk upon technology change

How it works

  • Business processes are defined in terms of Aggregates consuming messages and issuing events upon state change
  • Aggregates and Services are grouped in Modules
  • Modules are loaded into Pousse-Café's Runtime
  • Commands are submitted to the Runtime
  • The Runtime forwards the messages to Aggregates or their Factory or Repository
  • Aggregates issue Domain Events
  • Domain Events can cross Modules borders

See the reference guide for more information.

An Example

There is an example application available in GitHub. See README for instructions about how to run it and play with it. You can also just check-out the code.

Get started!

  1. Generate your first Pousse-Café project (do not forget to replace archetype version with the latest ): 

    mvn archetype:generate -B \
      -DarchetypeGroupId=org.pousse-cafe-framework \
      -DarchetypeArtifactId=pousse-cafe-sample-app-archetype \
      -DarchetypeVersion=${pousse-cafe.version} \
      -DgroupId=test \
      -DartifactId=test \
      -Dversion=0.0.0-SNAPSHOT
  2. Open the project in your favorite IDE
  3. Explore and execute the tests
